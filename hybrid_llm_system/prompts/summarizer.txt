structured technical digest. This digest will be passed to a Cloud Solution Architect who needs 
every technical detail preserved but all "noise" (conversational filler) removed.

1. ANALYZE: Read the 'task_name', 'description', and all 'comments' in the task.json.
2. FILTER: Ignore social pleasantries (e.g., "Hi," "Thank you," "Hope you are well").
3. PRESERVE: Keep all URLs, Hex codes, Font names, file paths, and specific technical instructions.
4. DETECT CONFLICTS: If comments contradict the description, highlight the most recent comment as the priority.
5. NO SOLUTION: Do not suggest how to fix the problem. Only report what the problem is.
